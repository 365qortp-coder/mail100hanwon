# 비보(VIBO) 브레인스토밍 대화 로그

> 이 문서는 기획자(사용자)와 AI가 나눈 브레인스토밍 전 과정을 구조화해 보존한 것이다.
> 원본 아이디어의 뉘앙스와 결정 맥락을 잃지 않기 위해, 사용자의 raw 발언을 최대한 살려 정리했다.
> 정제된 결론은 `비전보드앱_사업기획서.md` / `이루에이전트_스펙.md` / `경쟁앱_리서치.md` 참조.

## 세션 1 — 최초 아이디어 덤프

**사용자 원문 아이디어:**
- 비전보드인데 원하는 사진에 내 얼굴 합성해주고, 핸드폰에서 엄청 잘 보임 — 잠금화면/배경화면.
- 『It Works(잇 웍스)』(책 이름) 방식: 이름 쓰고, 한 줄 상황 + 날짜 써놓을 수 있음.
- 서로 자랑할 수 있게 — 오늘의집같이 — 자랑한 거 따라할 수 있게. 이런 바이럴 요소 잔뜩 넣기.
- 문제는 맘에 드는 사진 — 무료로 재가공 가능한 소스.
- 얼굴사진 합성을 위한 기획, 보안 정책 등.
- 비전보드 쓰고 실제 잘된 케이스 간증할 수 있게 — 바이럴 요소.
- 사용자 많아지고 → 사진 서로 제공하고 → 서로 자랑·격려 네트워크 전략.
- 볼링핀 전략 기획 (예: 리센느가 음방 1위 대박 터지는 거, 매일 비전보드 했다).
- "비전보드 했다"가 아니라 대명사/이름으로 "OO했다 — 이거 비보했다" 식.
- 사업 구조화.

**도출된 결정:**
- 제품 코어 4단: 만들기(얼굴합성+It Works 양식) → 매일 보기(잠금화면) → 자랑(오늘의집형) → 간증(비포/애프터 카드).
- 네이밍: **비보(VIBO)** — "비보했다" 동사화. 悲報 동음이의는 밈으로 역이용.
- 사진 소스 4계층: 자체 AI 라이브러리 우선 → 무료 스톡(Unsplash는 대량수집 금지라 부적합) → UGC 마켓 → 유저 직접 업로드.
- 얼굴합성 5원칙: 본인 얼굴만·셀럽 차단·AI 표시·미성년 보호·NSFW 차단. 데이터는 민감정보(생체) 취급.
- 볼링핀 헤드핀: 아이돌 지망생/오디션 준비생 + 팬덤 → 수험생 → 다이어트 → 재테크 → 해외.

## 세션 2 — 투두리스트 통합 & 사명/비전

**사용자 원문 아이디어:**
- 투두리스트도 넣기 — 오늘 당장 할 일 / 이번 달 / 이번 분기 / 올해 할 일.
- 이거 넣고 매일 접속하게 하기. 화면 = 사진 + 이뤄진 날짜 + 투두리스트.
- 사람들이 데드라인으로 느껴서 어렵다 생각할 수 있는데, 괜찮다고 해줘 — 다시 날짜 수정하고 투두리스트 수정하면 된다고 안심시켜줘.
- 비전보드 + 투두리스트 합친 거임.
- **사명**: 모두가 모두의 꿈을 이룰 수 있는, 가장 현실적인 방법으로 도와준다.
- **비전**: 매일 볼 수 있고 시각화된 걸로 세부적·실전적 계획을 세우고, 그걸 실현할 수 있는 앱 개발.
- 문해이루 에이전트(비전 읽고 계획 짜주는 AI)만 먼저 돌려봐.

**도출된 결정:**
- 4단 투두 + "매일 여는 이유"는 투두, "매일 보는 이유"는 비전보드로 역할 분리.
- **안심 UX**: 날짜·투두 무벌점 수정, "실패" 아니라 "경로 업데이트", 미완료 반복 시 벌점 대신 쪼개주기.
- **이루 에이전트** 스펙 작성 + 프로토타입 단독 실행(create/reschedule/stuck 3케이스) → 실측 판정 통과.

## 세션 3 — 카운셀링·얼굴인식·리서치

**사용자 원문 아이디어:**
- 실전계획 짜주는 거 말고, 현실적인 카운셀링 할 수 있게.
- ① 가치관 파악 — 질문 설문부터. 네 꿈이 진짜 네 꿈인지 파악부터. 데이팅 앱 '튤립'이 가치관 관련 질문 주는 거 — 그런 식 적용.
- ② 얼굴인식 가능? 은행 앱처럼 내 얼굴 상하좌우 인식되도록.
- AI가 가치관 기반으로 진짜 네가 원하는 꿈인지, 그 상황이 네가 원하는 상황 맞는지 같이 체크해줌.
- 투두리스트·비전보드 어플들 리서치.

**도출된 결정:**
- **§2.0 가치관 온보딩** 신설: 튤립식 "중요 질문 직접 선택"(리서치로 확인) + 썸원식 "매일 1문답"(누적 600만) 결합.
- 이루 에이전트에 **카운셀링 모드** 추가: 판정 금지, 질문으로만 거울. 대화 시뮬레이션 실측(케이스 D — 대기업 임원↔자유 가치관 충돌) 통과.
- **얼굴인식 확정**: ML Kit(무료, 고개 각도)로 상하좌우 챌린지 자체 구현(MVP) → 성장기 AWS Face Liveness($0.015/건). 딥페이크 완전 방어는 불가, 캐주얼 도용 억제용으로 규정.
- **경쟁 리서치**: "비전보드+AI투두+얼굴합성" 3종 결합 앱 부재. 한국 비전보드 전용 앱 공백. 최근접=MyFutureSelf.

## 세션 3-b — 이타 프레이밍 · AI 개입 · 듀오링고

**사용자 원문 아이디어:**
- 타인에게 좋은 영향을 주는 꿈이 더 잘 이뤄진다.
  - 아이돌: 좋은 음악·무대로 팬들에게 큰 기쁨 주고 나는 즐기고 결국 음방 1위.
  - 자영업: 진짜 맛있는 음식 주고 손님들 좋아해서 가게 대박.
  - 의사: 환자들 치료 잘 되고 병원 대박.
- (투두 쪼개주는 개입 등) 이런 건 반응 테스트 해봐야 함 — AI 개입당하는 거 싫어할 수 있음.
- 듀오링고처럼 — 비보 매일 볼 수 있게.

**도출된 결정:**
- **이타 프레이밍 원칙**(§2.1): 선언문에 "누가 기뻐하는지" 넣도록 유도. 공개해도 자랑 아닌 다짐으로 읽혀 공유 장벽↓.
- **AI 개입 거부감 리스크** 신설: 반응 테스트 최우선, 모든 개입은 제안+수락(거절 버튼·개입 끄기), 문구·빈도 A/B.
- **듀오링고식 매일 접속**: 스트릭+프리즈("쉼도 경로"), 하루 1개 마이크로 액션(투두/문답/응시 택1), 다정한 리마인더 캐릭터(죄책감 유발 금지로 차별화).

## 세션 4 — 저장소 분리 & 바이럴 훅

**사용자 원문 아이디어:**
- 지금 업로드 깃허브 vibo 프로젝트 새로 만들어서 하고 싶고 — 클로드 코워크에서 하면 핸드폰에서 언제든 할 수 있어?
- (저장소) 비공개 + 모든 대화 내용까지 아주 싹 다 전부 저장.
- 바이럴 돌릴 수 있게 — AI 사진 만든 거 공유하고 → "비보에서 했다" → "나도 해보기" → 이런 식으로.

**도출된 결정:**
- 별도 **vibo 저장소(Private)** 신설, 기획 문서 + 이 대화 로그 전부 이관.
- **핵심 바이럴 훅**(§6.1): 공유물마다 "비보에서 했다" 서명 + "나도 해보기" 원탭 딥링크 → 그 장면 세팅된 채 생성 화면 직행 → 순환.
- 코워크=핸드폰에서 claude.ai/code 접속해 언제든 이어작업, 단 작업물은 GitHub에 커밋돼 있어야 유지.
